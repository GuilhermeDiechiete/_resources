O que é MVC?

É acronimo de Model View Controllerrrrrrrrrrrrrrrrrrrrrrrrrrrr
É um padrão de arquitetura de software, que pode deixar nossas aplicações mais organizadas
A aplicação é dividida em camadas, cada sua responsabilidade
Teremos uma nova estrutura de arquivos e pastas
E a aplicação obedece um novo fluxo, que se repete para todas as ações 
Aplicação em MVC tendem a ter uma manutenção/evolução mais facil.

Camada do Modelo (Model)
    
    É uma camada onde vamos se interagir com o banco de dados
    Normalmente interage com os arquivos do controller 
    Responsavel por resgatar, atualizar, remover e criar dados 
    É comum que cada tabela seja um Model, assim como fazemos com o setup do Sequelize
    Os models são quem controlam a arquitetura do sistema, é facil entender a regra de negocios analisando eles.

Camada de Visualização (View)

    É onde apresentamos os dados que estão no banco de dados 
    Geralmente a view interage com o Controller, que é o meio de campo
    E tambem nas views temos a interação com o usuario, como formularios para inserir dados no sistema 
    Normalmente a exibição é feita atraves do HTML

Camada de controle ( Controller )

    É onde temos a interação entre o Model e View
    Podemos definir qual view sera impressa, processar dados que foram enviados para o banco ou para a view
    Os controllers terão um codigo parecido com os das rotas.


ESTRUTURA 

    controllers: pasta com os arquivos de controle 
    models: pasta que ficam os arquivos de model 
    views: arquivos da views
    routes: onde fica as rotas
    index.js: arquivo que inicia a aplicação
